构建一个 `chunk` 技术实践的实验平台
在相同文件、相同 `embedding`、相同检索流程的前提下，对比不同 `chunk` 策略对 `RAG` 系统命中率（回答质量）的影响。

比如通过以下操作：
1. 通过 `cli_fileUpload` 上传资源文件；
2. 对已经上传的文件分别执行不同的 `chunk` 技术实践类型（`cli_chunk_RCTS`、`cli_chunk_SC`、`cli_chunk_JE`），比如：调用 `cli_chunk_SC` 
3. 通过 `cli_search` 获取大模型的回答（经过向量查询）

基本职能是：
1. 文件上传(`cli_fileUpload`)：使用 `shell cli` 命令行方式指定一个文件进行上传处理（目标：`minio` 目录：`\test\uuid\`）
2. 文件 `chunk` 处理：每个 `python` 类实现一种 `chunk` 技术实践
以下技术实践类型：
2.1 传统切分(`cli_chunk_RCTS`)（实现eg: from langchain.text_splitter import RecursiveCharacterTextSplitter）
	切分完后将切分文本与该文本对应的向量保存数据库表 `t_chunk`
2.2 语义切分(`cli_chunk_SC`) (实现eg: 
		text_splitter = SemanticChunker(
			embeddings=self.embedding_service.embeddings, 
			breakpoint_threshold_type="percentile", 
			sentence_split_regex=split_regex, 
			min_chunk_size=min_chunk_size,
			add_start_index=True
		))
2.3 后置切块(`cli_chunk_JE`) (实现eg: 在 `LangChain` 中，通过 `JinaEmbeddings` 类配合特定的 `pooling` 策略实现)
3. 信息检索(`cli_search`)：以文件id、技术实践类型、用户提示词对向量数据进行检索查询，并将查询结果投喂大模型，最终将大模型回答返回（显示在终端）

表结构定义（拟定）:

t_document(
	id Column(String(50)),  # 文件ID (uuid)
	file_name Column(String(1024), nullable=False),  # 文件名（带扩展名）
	file_size Column(BigInteger),  # 文件大小
	file_etag Column(String(255)),
	file_type Column(String(100)),
	mime_type Column(String(100)),
	markdown_buffer
	description = Column(Text)
	);

t_chunk(
	id Column(String(50)),  # chunkID (uuid)
	document_id Column(String(50)),  # 文件ID (uuid)
	chunk_type,	# 标明是哪种策略（RCTS / SC / JE）
	raw_content, 	# 原始切分结果
	content,    # 可后处理的文本（当前阶段可等于 raw_content）
	chunk_index,	# 块在文档中的顺序
	start_position, # 在原文件中的字符偏移-开始位置
	end_position,	# 在原文件中的字符偏移-结束位置
	embedding_model = Column(String(255)),  # Track which model generated this embedding
	embedding_1536 = Column(Vector(1536)),  # Stored as for titan-v1, ada, openai-small
	embedding_1024 = Column(Vector(1024)),  # Stored as titan-v2-1024
	embedding_512 = Column(Vector(512)),  # Stored as for titan-v2-512
	embedding_256 = Column(Vector(256)),  # Stored as titan-v2-256
	embedding_3072 = Column(Vector(3072)),  # Stored as openai-large
	)
