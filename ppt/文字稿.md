了解什么是全文 Embedding，就先简单看懂它的原理
首先对句子进行分词器处理，得到全文 Token 序列，以此为基础
1.Attention注意力分配：对分词器中的每一个词(token)进行关注程度计算（计算该词对序列中所有词（包括前面和后面）的关注程度）
2.向量v_：对这些关注度进行加权求和，得到结果向量

解释：
Attention关注度计算与加权求和构成了全文 Embedding 的数字基础，模型对语言语法句法的学习程度会影响全文 Embedding的质量（比如相较于国外模型，国内模型对中文RAG系统更具有优势）
因为“模型学的是语法/句法功能，不是语义相似度”
当然 Tokenizer 分词阶段同样受到模型语言能力的影响。

最终会得到 Token 级向量序列，这样就得到了全文 Embedding
所以需要关注这句话：每个 token 能看到全文所有 token（包括前后的）
就是说，通过 token 向量可以知道与其相关的其他 token
